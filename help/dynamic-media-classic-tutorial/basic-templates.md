# 基本模板简介 {#basic-templates}

在Dynamic Media Classic术语中，模板是一个文档，在模板发布后，可以通过URL动态更改。 动态媒体经典优惠基本模板，从图像服务器调用的基于图像的模板，由图像和渲染的文本组成。

模板最强大的一个方面是它们具有直接集成点，使您可以将它们绑定到数据库。 因此，您不仅可以提供一个图像并调整其大小，还可以查询数据库来查找新的或销售的项目，并使其显示为图像上的叠加。 您可以要求提供项目的说明，并在您选择和布局的字体中将其显示为标签。 无限可能。

基本模板可以采用从简单到复杂的多种不同方式实现。 例如：

- 基本销售。 如果产品有免费送货，则使用“免费送货”等标签。 这些标签由Photoshop的商品团队设置，Web使用逻辑来知道何时将它们应用到图像。
- 高级销售。 每个模板具有多个变量，并且可能同时显示多个选项。 使用数据库、库存和业务规则确定何时将产品显示为“Just In”、“Clearance”或“Sold Out”。 还可以在产品后面使用透明度，在不同的背景（如不同的房间）上显示它。 相同的模板和／或资产可重新用于产品详细信息页面，以在不同背景上显示同一产品的更大或可缩放版本。

了解动态媒体经典仅提供这些基于模板的应用程序的可视部分非常重要。 动态媒体经典公司或其集成合作伙伴必须提供业务规则、数据库和开发技能来创建应用程序。 没有“内置”模板应用程序；设计人员在Dynamic Media Classic中设置模板，开发人员使用URL调用来更改模板中的变量。

在本教程的本节结尾，您将了解如何：

- 将PhotoshopPSD上传到Dynamic Media Classic以将其用作模板的基础。
- 创建由图像层组成的简单推销基本模板。
- 添加文本图层并通过使用参数使其变量。
- 构建模板URL并通过Web浏览器动态地处理图像。

## 基本模板概述

基本模板的定义（简称“模板”）是可URL寻址的分层图像。 最终结果是一幅图像，但URL可以更改它。 它可以由照片、文本或图形组成— Dynamic Media Classic中的P-TIFF资源的任意组合。

模板与PhotoshopPSD文件最相似，因为它们具有相似的工作流程和类似的功能。

- 这两层都由类似醋酸层叠的层组成。 您可以合成部分透明的图像，并通过图层的透明区域查看下面的图层。
- 可以移动和旋转图层以重新确定内容的位置，并可以更改不透明度和混合模式以使内容部分透明。
- 您可以创建基于文本的图层。 图像服务器使用的文本引擎与Photoshop和Illustrator相同，因此质量可能非常高。
- 可以将简单的图层样式应用于每个图层以创建特殊效果，如投影或发光。

但是，与PhotoshopPSD不同，图层可以完全动态并通过图像服务器上的URL进行控制。

- 您可以向所有模板属性添加变量，从而轻松地动态更改其合成。
- 名为参数的变量允许您仅公开要更改的模板部分。

您只需为每个将发生变化的图层添加一个占位符，而不是像在Photoshop那样将所有图层放在一个文件中并显示和隐藏它们（尽管您也可以这样做，如果愿意）。

使用占位符，您可以动态地将某个图层的内容替换为另一个已发布的资产，并且该资产将自动采用该图层所替换的图层的相同属性（如大小和旋转）。

由于基本模板通常在Photoshop设计，但通过URL进行部署，因此模板项目既需要设计技能，也需要技术技能。 我们通常假定创作模板的人员是Photoshop设计人员，而实施模板的人员是Web开发人员。 创意团队和开发团队必须密切合作，才能使模板取得成功。

模板项目可能相对简单，也可能非常复杂，具体取决于业务规则和应用程序的需要。 基本模板是从图像服务器调用的，但是，由于Dynamic Media Classic环境的灵活性，您甚至可以将模板嵌套在其他模板中，这样您就可以创建出相当复杂的图像，这些图像可以通过常用的变量进行链接。

## 创建基本模板

使用基本模板时，通常按照下图中的工作流步骤操作。 如果您使用动态文本图层，则标记有虚线的步骤是可选的，并在下面的说明中指示为“文本工作流”。 如果不使用文本，则只遵循主路径。

![图像](assets/basic-templates/basic-templates-1.png)

_基本模板工作流。_

1. 设计和创建资源。 大多数用户在Adobe Photoshop都这样做。 以您需要的确切大小设计资源——如果缩略图页面的图像大小为200像素，则以200像素进行设计。 如果需要放大它，设计大小约为2000像素。 使用Photoshop(和／或另存为位图的Illustrator)创建资源，使用Dynamic Media Classic将各个部分组合到一起，管理图层并添加变量。
2. 设计图形资源后，将其上传到Dynamic Media Classic。 我们建议您上传整个分层的PSD文件，并让Dynamic Media Classic通过在上传时使用“维护图层”选项，为每个图层创建一个文件，而 **不是从PSD上传单个资产** （有关详细信息，请参阅下文）。 _文本工作流：如果创建动态文本，也请上传字体。 动态文本是变量，并通过URL进行控制。 如果您的文本是静态的，或只有几个短语没有更改（例如，标记说“新建”或“销售”，而不是“X% Off”,X是变量数），我们建议在Photoshop预渲染文本，并将栅格化图层上传为图像。 这将更轻松，您可以根据需要完全设置文本样式。_
3. 使用“构建”菜单的“模板基础”编辑器在Dynamic Media Classic中构建模板，并添加图像图层。 文本工作流：在同一编辑器中创建文本图层。 在Dynamic Media Classic中手动构建模板时需要此步骤。 选择与您的设计匹配的画布大小，将图像拖放到画布上，并设置图层属性（大小、旋转、不透明度等）。 您不会将每个可能的图层放在模板上，而只是每个图像图层一个占位符。 _文本工作流：使用文本工具创建文本图层，与在Photoshop创建文本图层类似。 您可以使用Photoshop文字工具提供的相同选项选择字体并设置样式。_ 另一个工作流程是上传PSD并让Dynamic Media Classic生成“免费”模板，甚至可以重新创建文本图层。 稍后将详细讨论此问题。
4. 创建图层后，请向要通过URL控制的任何图层的任何属性添加参数（变量），包括该图层的源（图像本身）。 _文本工作流：您还可以向文本图层添加参数，既可以控制文本内容、图层本身的大小和位置，也可以控制所有格式选项，如字体颜色、字体大小、水平跟踪等。_
5. 创建与模板大小匹配的图像预设。 我们建议这样做，以便始终以1:1大小调用模板，并为调整大小以适应模板的任何大型图像图层添加锐化功能。 如果要构建要放大的模板，则不必执行此步骤。
6. 发布，从Dynamic Media Classic预览复制URL，并在浏览器中测试它。

## 准备模板资产并将其上传到Dynamic Media Classic

在将模板资产上传到Dynamic Media Classic之前，您需要完成一些准备步骤。

### 准备要上传的PSD

在将您的Photoshop文件上传到Dynamic Media Classic之前，请简化Photoshop的图层，使其更易于使用并与图像服务器实现最大兼容。 您的PSD文件通常由许多Dynamic Media Classic无法识别的元素组成，您最终可能还会遇到许多难以管理的小部分。 请务必保存主控PSD的备份，以备以后编辑原始PSD时使用。 您将上传简化副本，而不是主控。

![图像](assets/basic-templates/basic-templates-2.jpg)

1. 通过将需要打开／关闭的相关图层合并／拼合为单个图层，简化图层结构。 例如，标签“NEW”和蓝色横幅将合并到单个图层中，这样您只需单击一下即可显示或隐藏它们。
   ![图像](assets/basic-templates/basic-templates-3.jpg)
2. Dynamic Media Classic或图像服务器不支持某些图层类型和图层效果，在上传之前需要进行栅格化。 否则，可能会忽略效果，或丢弃图层。 栅格化图层意味着从可编辑到不可编辑进行转换。 要栅格化图层效果或文本图层，请创建空图层，同时选择并使用“图层” **>“合并图层”** 或CTRL + E/CMD + E进行合并。

   - Dynamic Media Classic不能对图层进行分组或链接。 组或链接集中的所有图层都将转换为不再分组／链接的单独图层。
   - 上传时，图层蒙版将转换为透明度。
   - 不支持调整图层，将丢弃调整图层。
   - 填充图层（如纯色图层）将栅格化。
   - 在上传时，智能对象图层和矢量图层将栅格化为普通图像，并应用和栅格化智能过滤器。
   - 除非使用“提取文本”选项，否则文本图层也将进行栅格化——有关详细信息，请参见下文。
   - 大多数图层效果将被忽略，只支持少数混合模式。 如果有疑问，请在Dynamic Media Classic中添加简单效果（如内阴影或投影、内光或外光），或使用空白图层合并和栅格化Photoshop的效果。

### 使用字体

如果需要生成动态文本，您还将上传和发布您的字体。 Dynamic Media Classic附带的唯一字体是Arial。

每个公司都有责任获取在Web上使用字体的许可——只要在您的计算机上安装字体，您就无权在Web上以商业方式使用它，并且如果未经许可而使用，您的公司可能会面临字体发布者的法律诉讼。 此外，许可条款也会有所不同——例如，您可能需要单独的许可才能进行打印和屏幕显示。

Dynamic Media Classic支持标准OpenType(OTF)、TrueType(TTF)和Type 1 Postscript字体。 不支持仅限Mac的手提箱字体、文字集合文件、Windows系统字体和专有机器字体（如雕刻机或刺绣机使用的字体）-您必须将它们转换为一种标准字体格式，或替换在Dynamic Media Classic和图像服务器中使用的类似字体。

在字体上传到Dynamic Media Classic（与任何其他资产一样）后，还必须将其发布到图像服务器。 一个非常常见的模板错误是忘记发布字体，这会导致图像错误——图像服务器不会用其他字体替换它的位置。 此外，如果要在上传时使用“ **提取文本** ”选项，则必须先上传字体文件，然后再上传使用这些字体的PSD。 提 **取文本** 功能将尝试将文本重新创建为可编辑的文本图层，并将其放在Dynamic Media Classic模板中。 下一个主题PSD选项中将讨论此问题。

请注意，字体具有多个内部名称，这些名称通常与其外部文件名不同。 您可以在Dynamic Media Classic中查看该资产的“详细信息”页面上的所有不同名称。 以下是Dynamic Media Classic的“元数据”选项卡下列出的字体“Adobe Caslon·Pro半粗体”的名称：

![图像](assets/basic-templates/basic-templates-4.jpg)

_Dynamic Media Classic中字体的“详细信息”页面上的元数据选项卡。_

Dynamic Media Classic使用此字体的文件名(ACaslonPro-Semibold)作为其资产ID，但该名称不是模板使用的名称。 模板使用底部列出的富文本格式(RTF)名称。 RTF是图像服务器文本引擎的本机“语言”。

如果您需要通过URL更改字体，则必须调用字体的RTF名称（而非资产ID），否则您将收到错误。 在这种情况下，此字体的正确名称为“Adobe CaslonPro”。 我们将在下面主题RTF和文本参数中讨论有关字体和RTF的更多信息。

在Windows和Mac系统上，最常见的字体文件格式是OpenType和TrueType。 OpenType的扩展名为。OTF，而TrueType的扩展名为。TTF。 这两种格式在Dynamic Media Classic中同样有效。

### 上传PSD时选择选项

您无需上传Photoshop文件(PSD)即可创建模板；可以在Dynamic Media Classic中使用任何图像资产构建模板。 但是，上传PSD可以简化创作，因为通常，这些资产已包含在分层的PSD中。 此外，Dynamic Media Classic将在您上传图层式PSD时自动生成模板。

- **保持图层。** 这是最重要的选择。 这告诉Dynamic Media Classic为每个Photoshop图层创建一个图像资源。 如果未选中，则会禁用所有其他选项，并将PSD拼合为单个图像。
- **创建****模板。** 此选项将选取各个生成的图层，并通过将它们组合在一起自动创建模板。 使用自动生成的模板的一个缺点是Dynamic Media Classic将所有图层放在一个文件中，而我们只需要每个图层一个占位符。 删除额外的图层非常简单，但如果您有许多图层，则重新创建它们会更快。 请务必重命名新模板；如果不这样做，下次重新上传同一PSD时，该PSD将被覆盖。
- **提取文本。** 这会使用您上传的字体将PSD中的文本图层重新创建为模板中的文本图层。 如果文本位于Photoshop的路径中，并且要在模板中保留该路径，则需要执行此步骤。 此功能要求您使用“创 **建模板** ”选项，因为提取的文本只能由上传时生成的模板创建。
- **将图层扩展到背景大小。** 此设置使每个图层的大小与整个PSD画布相同。 这对于始终保持固定位置的图层非常有用：否则，在将图像交换到同一图层时，可能必须重新确定它们的位置。
- **图层命名。** 这告诉Dynamic Media Classic如何命名每个图层生成的每个资产。 我们建议使用 **Photoshop****和层名** ，或 **者使** 用Photoshop和 **层******,或者使用号。 这两个选项都使用PSD名称作为名称的第一部分，并在末尾添加图层名称或编号。 例如，如果您有一个名为“shirt.psd”的PSD，并且它有名为“front”、“sives”和“cillar”的图层，如果您使用“ **Photoshop”和** Layer Name **** ”选项上传，Dynamic Media Classic将生成资产ID“shirt_front”、“shit_colews”和“colarllar”。 使用其中一个选项有助于确保名称在Dynamic Media Classic中是唯一的。

## 创建带有图像图层的模板

即使Dynamic Media Classic可以从分层的PSD自动创建模板，您也应知道如何手动构建模板。 如上所述，有时您不想使用Dynamic Media Classic创建的模板。

### 模板基础UI

我们先熟悉一下编辑界面。

左中心是您的工作区，显示最终模板的预览。 右侧是“图层”和“图层属性”面板。 这些区域是您最擅长的工作。

![图像](assets/basic-templates/basic-templates-5.jpg)

_构建模板基础知识页面。_

- **预览/工作区。** 这是主窗。 您可以在此使用鼠标移动、调整图层大小和旋转图层。 图层轮廓显示为虚线。
- **图层。** 这与Photoshop图层面板相似。 在向模板添加图层时，这些图层将显示在此处。 图层从上到下堆叠-“图层”面板中的顶层在列表中显示在其下的其他层上方。
- **图层属性。** 您可以在此使用数字控件调整图层的所有属性。 首先选择一个图层，然后调整其属性。
- **复合****URL。** UI底部是复合URL区域。 本教程的本节将不讨论此问题，但您将看到模板被解构为一系列图像服务URL修饰符。 此区域是可编辑的——如果您非常熟悉图像服务器命令，则可以在此处手动编辑模板。 但是，您也可以打破它。 与Photoshop一样，图层编号为0的开始。 画布为0层，您自己添加的第一个层为1层。 混合模式确定图层的像素如何与其下方的像素混合。 您可以使用混合模式创建各种特殊效果。

#### 使用模板基础编辑器

以下是开始基本模板的工作流步骤：

1. 在Dynamic Media Classic中，转至“构建”>“ **模板基础知识”**。 您可以不选择任何内容，也可以通过选择图像来开始，图像将成为模板的第一层。
2. 选择“Size（大小）” ，然后 **按“OK(确定**)”。 这个尺寸应该与你在Photoshop设计的尺寸相符。 将加载模板编辑器。
3. 如果在第1步中未选择图像，则在左侧的“资产”面板中搜索或浏览到图像，然后将其拖动到工作区域。

   - 图像将自动调整大小以适合画布的大小。 如果您计划替换高分辨率图像，则通常会导入一张大(2000 px)P-TIFF图像并将其用作占位符。
   - 这应是模板的最底层图层，但您以后可以对图层重新排序。

4. 直接在工作区中或通过调整“图层属性”面板中的设置来调整图层大小或位置。
5. 根据需要拖入其他图像图层。 根据需要添加图层效果。 请参阅下 _面的添加图_&#x200B;层效果主题。
6. 单击 **“保存**”，选择一个位置，然后为模板指定一个名称。 您可以预览，但此时您的模板将看起来与拼合的Photoshop图像完全一样——它尚不可更改。

### 添加图层效果

图像服务器支持一些程序化图层效果——可更改图层内容外观的特殊效果。 它们的工作方式与Photoshop的图层效果类似。 它们附接到层，但独立于层进行控制。 您无需对图层本身进行永久更改即可调整或删除它们。

- **投影**。 在图层边界外应用阴影（由x和y像素偏移定位）。
- **内阴影**。 在图层边界内应用阴影（由x和y像素偏移定位）。
- **外发光**。 在图层的所有边缘周围均匀地应用发光效果。
- **内发光**。 均匀地在图层的所有边缘内应用发光效果。

![图像](assets/basic-templates/basic-templates-6.jpg)

_具有和没有投影的图层_

要添加效果，请单击“ **添加效果**”，然后从菜单中选择效果。 与普通图层一样，您可以在“图层”面板中选择效果，然后使用“图层属性”面板调整其设置。

阴影效果水平或垂直偏移离图层，而发光效果在所有方向均匀应用。 内部效果作用在图层的不透明部分的顶部，而外部效果仅影响透明区域。

进一步了[解添加图层效果](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/creating-template.html#using-shadow-and-glow-effects-on-layers)。

### 添加参数

如果您只是组合图层并保存它们，则最终结果与拼合的Photoshop图像无异。 模板的特殊之处在于能够向每个层的属性添加参数，以便通过URL动态更改它们。

在Dynamic Media Classic术语中，参数是可链接到模板属性的变量，因此可以通过URL操纵它。 当您向图层添加参数时，Dynamic Media Classic会在URL中显示该属性，方法是在参数名称前加上美元符号($)-例如，如果您创建一个名为“size”的参数来更改图层的大小，Dynamic Media Classic会将您的参数重命名为$size。

如果不为属性添加参数，该属性将保持隐藏在Dynamic Media Classic数据库中，而不会显示在URL中。

![图像](assets/basic-templates/parameters.png)

如果没有参数，URL通常会更长一些，尤其是如果您也使用动态文本。 文本会在每个URL上添加许多额外字符。

最后，初始参数集将成为模板中属性的默认值。 如果您构建模板、添加参数，然后调用不带参数的URL，图像服务器将创建包含您在模板中保存的所有默认值的图像。 仅当要更改属性时才需要参数。 如果属性不需要更改，则无需设置参数。

#### 创建参数

这是创建参数的工作流：

1. 单击 **要** 为其创建参数的层名称旁边的“参数”按钮。 此时将打开“参数”屏幕。 它列表层上的每个属性及其值。
1. 选择 **要作** 为参数的每个属性名称旁的“开”选项。 将显示默认参数名称。 您只能向从默认状态更改的属性添加参数。

   - 例如，如果添加一个图层并将其保留在默认的xy位置0,0,Dynamic Media Classic将不显示“位置” **属性** 。 要修复，请至少移动一个像素。 现在，Dynamic Media Classic将将 **Position** （位置）显示为可参数化的属性。
   - 要向show/hide属性添加参数（该属性打开和关闭图层），请单 **击** “显示”或 **“隐藏图层** ”图标以关闭图层（如果需要，可以随后将其重新打开）。 Dynamic Media Classic现在将公开可 **以参数化** 的“隐藏”属性。

1. 将默认参数名称重命名为在URL中更容易识别的名称。 例如，如果要添加参数以更改图像顶部的横幅图层，请将默认名称“layer_2_src”更改为“banner”。
1. 按“ **关闭** ”退出“参数”屏幕。
1. 通过单击“参数”按钮并添加和重 **命名参数** ，为其他图层重复此过程。
1. 完成后保存更改。

>[!TIP]
>
>将您的参数重命名为有意义的内容，并制定命名规范以标准化这些名称。 请确保设计和开发团队事先就命名约定达成一致。
>
>无法添加参数，因为您看不到该属性？ 只需更改图层的默认属性（通过移动、调整大小、隐藏等）。 您现在应该可以看到该属性。

进一步了 [解模板参数](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/creating-template-parameters.html)。

## 创建带有文本图层的模板

现在您将学习如何创建包含文本图层的基本模板。

### 了解动态文本

您现在了解如何使用图像图层构建基本模板。 对于许多应用程序，这是您所需要的。 正如您在上一个练习中所看到的，具有简单文本（如“Sale”和“New”）的图层可栅格化并作为图像处理，因为它们的文本无需更改。

但是，如果您需要：

- 添加一个标签以表示“25%关闭”，其值25%可变
- 在图像顶部添加带有产品名称的文本标签
- 根据模板所在的国家／地区，将图层本地化为不同语言

在这种情况下，您需要添加一些包含参数的动态文本图层以控制文本和／或格式。

要创建文本，您需要上传一些字体——否则，Dynamic Media Classic将默认为Arial。 字体还必须发布到图像服务器，否则当字体尝试呈现使用该字体的任何文本时，它将生成错误。

### RTF和文本参数

要使用模板基础工具向文本添加变量，您应了解文本的呈现方式。 图像服务器使用Adobe文本引擎生成文本，该引擎与Photoshop和Illustrator使用的引擎相同，并将其合成为最终图像中的一个图层。 为了与引擎通信，图像服务器使用富文本格式或RTF。

RTF是Microsoft开发的用于指定文档格式的文件格式规范。 它是大多数文字处理和电子邮件软件使用的标准标记语言。 如果您写入URL &amp;text=\b1 Hello，图像服务器将生成一个带粗体文字字“Hello”的图像，因为\b1是用于使文本变粗的RTF命令。

好消息是Dynamic Media Classic可为您生成RTF。 无论您何时在模板中键入文本并添加格式，Dynamic Media Classic都会自动将RTF代码静默地写入模板中。 我们之所以提到它，是因为您将直接向RTF中添加参数，因此您应该对它有点熟悉。

#### 创建文本图层

您可以通过以下两种方式在Dynamic Media Classic中的模板中创建文本图层：

1. Dynamic Media Classic中的文本工具。 下面我们将讨论此方法。 模板基础工具编辑器提供了一个工具，可让您创建文本框、输入文本和设置文本格式。 Dynamic Media Classic根据需要生成RTF并将其放在单独的图层中。
2. 提取文本（上传时）。 另一种方法是在Photoshop创建文本图层，并将其另存为普通文本图层（而不是将其栅格化为图像图层）。 然后，将文件上传到Dynamic Media Classic并使用“提 **取文本** ”选项。 Dynamic Media Classic将使用RTF命令将每个Photoshop文本层转换为图像服务文本层。 如果您使用此方法，请确保首先将字体上传到Dynamic Media Classic，否则Dynamic Media Classic将在上传时替换默认字体，并且无法轻松地重新替换正确的字体。

### 文本编辑器

使用文本编辑器输入文本。 文本编辑器是WYSIWYG界面，它允许您使用与Photoshop或Illustrator类似的格式控件输入文本并设置其格式。

![图像](assets/basic-templates/basic-templates-9.jpg)

_模板基础知识文本编辑器。_

您将在预览选项卡中完成大 **部分工** 作，该选项卡允许您输入文本并查看文本在模板中的显示效果。 还有一个“ **源** ”选项卡，用于根据需要手动编辑RTF。

常规工作流是使用 **预览** 选项卡键入一些文本。

然后选择文本，并使用顶部的控件选择一些格式，如字体颜色、字体大小或对齐。 文本按照您需要的方式设置样式后，单 **击** “应用”以在工作区预览中查看更新。 然后，关闭文本编辑器以返回到模板基础主窗口。

#### 使用文本编辑器

以下是在“模板基础知识”构建页面中添加文本的工作流步骤：

1. 单击 **构建页** 面顶部的“文本”工具按钮。
2. 拖出要显示文本的文本框。 文本编辑器窗口将在模态窗口中打开。 在后台，您将看到模板，但只有编辑完文本后，模板才可编辑。
3. 键入要在首次加载模板时显示的示例文本。 例如，如果您正在为个性化的电子邮件图像创建文本框，您的文本可能会显示“您好”。 现在是拯救的时候了！” 稍后，您将添加一个文本参数，以将Name替换为您在URL上发送的值。 在单击“应用”之前，您的文本不会显示在窗口下方的模 **板中**。
4. 要设置文本格式，请使用鼠标拖动来选择文本，然后在UI中选择格式控件。

   - 有许多格式选项。 一些最常见的字体（面部）、字体大小和字体颜色，以及左／中/右对齐。
   - 别忘了先选择文本。 否则，您将无法应用任何格式。
   - 要选择其他字体，请确保选择文本并打开“字体”菜单。 编辑器将显示上传到Dynamic Media Classic的所有字体的列表。 如果计算机上也安装了字体，它将显示为黑色。 如果计算机上未安装此组件，则显示为红色。 但是，单击“应用”时，它仍将在预览窗口中 **呈现**。 您只需将字体上传到Dynamic Media Classic，即可让任何使用Dynamic Media Classic的用户都能使用这些字体。 发布后，图像服务器将使用这些字体生成文本——您的用户无需安装任何字体即可查看您创建的文本，因为它是图像的一部分。
   - 与Photoshop和Illustrator不同，图像服务器可以在文本框中垂直对齐文本。 默认为顶部对齐。 要更改此项，请选择文本，然后从“垂 **直对齐** ” **菜单中选** 择“中间对齐”或 **“** 底部对齐”。
   - 如果文本过大，而框不能显示（或文本框太小），则其全部或部分内容将被剪辑并消失。 减小字体大小或增大框。

5. 单击 **“应用** ”，查看您所做的更改在工作区窗口中生效。 必须单击“ **应用**”，否则您将丢失所做的编辑。
6. 完成后，单击“关 **闭**”。 如果要返回编辑模式，请多次单击文本图层以重新打开文本编辑器。

如果您的系统中已本地安装字体，文本编辑器将准确预览字体的大小。

### 关于向文本图层添加参数

现在，我们按照类似的过程添加文本参数，就像我们对图层参数所做的那样。 文本层还可以采用图层参数来调整大小、位置等；但是，它们可以采用其他参数，以便您控制RTF的任何方面。

与层参数不同，您只选择要更改的值并向其添加参数，而不是向整个属性添加参数。

## 发布和创建模板URL

### 创建图像预设

为模板创建预设不是必需步骤。 我们建议将此作为最佳实践，以便始终以1:1大小调用模板，并为调整大小以适应模板的任何大型图像图层添加锐化功能。 如果调用的图像没有预设，图像服务器可以随意调整图像大小至默认大小（约400像素），而不会应用默认锐化。

模板的图像预设没有什么特别的。 如果您已经拥有相同大小的静态图像的预设，则可以改用它。

### 发布

您需要运行发布才能查看实时推送到图像服务器的更改。 记住需要发布的内容：各种图像资源图层、动态文本的字体以及模板本身。 与图像集和旋转集等其他Dynamic Media Classic富媒体资产相似，基本模板是人工构建——它是数据库中的一个行项，它使用一系列图像服务命令引用图像和字体。 因此，发布模板时，您所做的只是更新图像服务器上的数据。

了解有关发布 [模板的更多信息](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/publishing-templates.html)。

### 模板URL构建

基本模板与常规图像调用具有相同的基本URL语法，如前所述。 模板通常具有更多修饰符(由和号(&amp;)分隔的命令)，如带值的参数。 但主要区别在于您调用模板作为主图像，而不是调用静态图像。

![图像](assets/basic-templates/basic-templates-11.jpg)

与图像预设不同，在预设名称的每侧都有一个美元符号($)，参数的开头有一个美元符号。 这些美元信号的放置很重要。

**正确：**

`$text=46-inch LCD HDTV`

**不正确：**

`$text$=46-inch LCD HDTV`

`$text=46-inch LCD HDTV$`

`text=46-inch LCD HDTV`

如前所述，参数用于更改模板。 如果调用模板时没有参数，该模板将恢复为在模板基础创作工具中设计的默认设置。 如果属性不需要更改，则无需设置参数。

![image](assets/basic-templates/sandals-without-with-parameters.png)_不设置参数（上图）和参数（下图）的模板示例。_
