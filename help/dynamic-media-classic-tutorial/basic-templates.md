# 基本模板简介 {#basic-templates}

在Dynamic Media Classic术语中，模板是指可以在发布模板后通过URL动态更改的文档。 Dynamic Media Classic提供基本模板，即从图像服务器调用的基于图像的模板，由图像和渲染的文本组成。

模板最强大的一个方面是它们具有直接集成点，使您能够将它们绑定到数据库。 因此，您不仅可以提供图像并调整其大小，还可以查询数据库以查找新项目或销售项目，并使其在图像上显示为叠加。 您可以要求项目说明，并使其显示为所选字体及版面的标签。 这种可能性是无限的。

基本模板可以通过多种不同的方式实施，从简单到复杂。 例如：

- 基本推销。 如果该产品有免运费，则使用“免运费”等标签。 这些标签由Photoshop中的商品团队设置，Web使用逻辑来知道何时将它们应用于图像。
- 高级推销。 每个模板都有多个变量，并且可以同时显示多个选项。 使用数据库、库存和业务规则来确定何时将产品显示为“刚好进入”、“清除”或“售罄”。 也可以使用产品后面的透明度在不同背景中显示，例如在不同房间中。 相同的模板和/或资产可以在产品详细信息页面上重新调整用途，以在不同背景上显示相同产品的更大版本或可缩放版本。

请务必了解，Dynamic Media Classic仅提供这些基于模板的应用程序的可视部分。 Dynamic Media Classic公司或其集成合作伙伴必须提供创建应用程序的业务规则、数据库和开发技能。 没有“内置”模板应用程序；设计人员在Dynamic Media Classic中设置模板，开发人员使用URL调用更改模板中的变量。

在本教程的此部分结束时，您将知道如何：

- 将PhotoshopPSD上传到Dynamic Media Classic以用作模板的基础。
- 创建包含图像图层的简单促销基本模板。
- 添加文本图层，并使用参数使其可变。
- 构建模板URL并通过Web浏览器动态处理图像。

>
>本章中的所有URL仅用于说明目的；它们不是实时链接。

## 基本模板概述

基本模板（或简称“模板”）的定义是可通过URL寻址的分层图像。 最终结果是一个图像，但图像可以由URL更改。 它可以由照片、文本或图形组成，这些是Dynamic Media Classic中PTIFF资源的任意组合。

模板与PhotoshopPSD文件最相似，因为它们具有相似的工作流程和相似功能。

- 这两层都像栈叠的醋酸盐。 您可以复合部分透明图像，并查看从图层的透明区域到下面的图层。
- 可以移动和旋转图层以重新定位内容，并且可以改变不透明度和混合模式以使内容部分透明。
- 可以创建基于文本的图层。 质量可能非常高，因为图像服务器使用与Photoshop和Illustrator相同的文本引擎。
- 可以将简单的图层样式应用于每个图层以创建特殊效果，例如投影或光晕。

但是，与PhotoshopPSD不同，图层可以完全动态化，并通过Image Server上的URL进行控制。

- 您可以将变量添加到所有模板属性，以便快速更改其组成。
- 称为参数的变量允许您仅公开要更改的模板部分。

您只需为每个将发生变化的图层添加一个占位符，而不必像在Photoshop中那样将所有图层放在单个文件中，并且显示和隐藏它们（不过您也可以根据需要这样做）。

使用占位符，您可以动态地将图层的内容与另一个已发布的资产交换，并且它会自动采用被替换图层的相同属性（例如大小和旋转）。

由于基本模板通常在Photoshop中设计，但通过URL部署，因此模板项目需要混合使用设计和技术技能。 我们通常假定进行创意模板工作的人员是Photoshop设计人员，而实施模板的人员是Web开发人员。 为了使模板取得成功，创意团队和开发团队必须密切合作。

模板项目可能相对简单或极其复杂，具体取决于应用程序的业务规则和需求。 基础模板是从图像服务器调用的，但由于Dynamic Media Classic环境的灵活性，您甚至可以将模板嵌套在其他模板中，从而允许您创建可通过常用命名变量链接的相当复杂的图像。

- 详细了解 [模板基础知识](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/quick-start-template-basics.html).
- 了解如何创建 [基本模板](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/creating-template.html#creating_a_template).

## 创建基本模板

使用基本模板时，您通常遵循下图中的工作流步骤。 如果您使用动态文本图层，则标有虚线的步骤是可选的，下面的说明中将其指示为“文本工作流”。 如果不使用文本，请仅遵循主路径。

![图像](assets/basic-templates/basic-templates-1.png)

_基本模板工作流。_

1. 设计和创建您的资产。 大多数用户在Adobe Photoshop中都会这样做。 按照所需的确切大小设计资产 — 如果缩略图页面的图像为200像素，则将其设计为200像素。 如果您需要放大它，请将其设计成大约2000像素的大小。 使用Photoshop(和/或Illustrator另存为位图)创建资产，并使用Dynamic Media Classic将零件装配在一起、管理图层和添加变量。
2. 设计图形资源后，将其上传到Dynamic Media Classic。 我们建议您上传整个分层的PSD文件，并让Dynamic Media Classic使用PSD中的每层创建一个文件，而不是从Layer上传单个资源。 **保留图层** 上传选项（有关更多详细信息，请参阅下文。） _文本工作流：如果创建动态文本，则也要上载字体。 动态文本是可变的，可通过URL控制。 如果您的文本是静态的或只有几个简短短语不会更改 — 例如，标记上写着“New”或“Sale”，而不是“X% Off”，其中X是一个可变数字 — 我们建议在Photoshop中预呈现文本，并以栅格化的图层作为图像上传。 这样会更简单，而且您可以完全按照自己的需要来设置文本样式。_
3. 使用“生成”菜单的“模板基础知识”编辑器在Dynamic Media Classic中生成模板并添加图像图层。 文本工作流：在同一编辑器中创建文本图层。 在Dynamic Media Classic中手动构建模板时，需要执行此步骤。 选择与您的设计匹配的画布大小，将图像拖放到画布上，并设置图层属性（大小、旋转、不透明度等）。 您不是在模板上放置所有可能的图层，而是只为每个图像图层放置一个占位符。 _文本工作流：使用“文本”工具创建文本图层，与在Photoshop中创建文本图层类似。 可以使用Photoshop文字工具提供的相同选项来选择字体和样式。_ 另一个工作流程是上传PSD并让Dynamic Media Classic生成“自由”模板，甚至可以重新创建文本图层。 稍后将对此进行更详细的讨论。
4. 创建图层后，将参数（变量）添加到要通过URL控制的任何图层的任何属性中，包括图层的源（图像本身）。 _文本工作流：您还可以向文本图层添加参数，以控制文本的内容和图层本身的大小和位置，以及所有格式选项，如字体颜色、字体大小、水平跟踪等。_
5. 创建与模板大小匹配的图像预设。 我们建议这样做，以便始终以1:1的大小调用模板，并且还要为任何调整大小以适合模板的大型图像图层添加锐化。 如果您正在构建要缩放的模板，则无需执行此步骤。
6. 发布，从Dynamic Media Classic预览中复制URL，然后在浏览器中对其进行测试。

## 准备模板资源并将其上传到Dynamic Media Classic

在将模板资产上传到Dynamic Media Classic之前，您需要完成一些准备步骤。

### 准备PSD以供上传

在将Photoshop文件上传到Dynamic Media Classic之前，请简化Photoshop中的图层，使其更易于使用，并且与Image Server具有最大的兼容性。 您的PSD文件通常包含许多Dynamic Media Classic无法识别的元素，并且最终可能包含许多难以管理的小片段。 请务必保存主控PSD的备份，以防以后需要编辑原始内容。 您将上传简化的副本，而不是主控。

![图像](assets/basic-templates/basic-templates-2.jpg)

1. 通过将需要一起打开/关闭的相关层合并/拼合到单个层来简化层结构。 例如，标签“NEW”和蓝色横幅将合并到单个图层中，以便您只需单击一下即可显示或隐藏它们。
   ![图像](assets/basic-templates/basic-templates-3.jpg)
2. Dynamic Media Classic或Image Server不支持某些图层类型和图层效果，因此需要在上传之前栅格化。 否则，可能会忽略影响或丢弃图层。 栅格化图层是指将是否可以编辑转换为不可编辑。 要栅格化图层效果或文本图层，请创建一个空图层，然后同时选择这两个图层并使用以下方式合并 **“图层”>“合并图层”** 或CTRL + E/CMD + E。

   - Dynamic Media Classic无法对图层进行分组或链接。 组或链接集中的所有图层都转换为不再分组/链接的单独图层。
   - 上传时，图层蒙版会转换为透明度。
   - 不支持调整图层，因此将放弃调整图层。
   - 填充图层（例如“纯色”图层）将被栅格化。
   - 智能对象层和矢量层在上传时栅格化为普通图像，并应用和栅格化智能滤镜。
   - 除非使用“提取文本”选项，否则文本图层也将栅格化 — 请参阅以下内容以了解更多信息。
   - 大多数图层效果都将被忽略，仅支持少数混合模式。 如有疑问，请在Dynamic Media Classic中添加简单效果（例如内部或投影、内部或外部光晕），或使用空白图层在Photoshop中合并并栅格化效果。

### 使用字体

如果需要生成动态文本，您还将上载和发布字体。 Dynamic Media Classic中唯一包含的字体是Arial。

每家公司都有责任获得使用网络字体的许可 — 只是将一种字体安装在您的计算机上，并不能让您在网络上获得商业使用权，而且如果没有获得许可，您的公司可能会面临字体发布者的法律诉讼。 此外，许可条款各不相同 — 例如，您可能需要单独的打印与屏幕显示许可证。

Dynamic Media Classic支持标准OpenType(OTF)、TrueType (TTF)和Type 1 Postscript字体。 Mac — 仅手提箱字体、文字收藏集文件、Windows系统字体和专用计算机字体（如雕刻或刺绣机使用的字体）均不受支持 — 您必须将它们转换为标准字体格式之一，或替换要在Dynamic Media Classic和Image Server中使用的类似字体。

字体像任何其他资源一样，上传到Dynamic Media Classic后，还必须将其发布到图像服务器。 一个非常常见的模板错误是忘记发布字体，这将导致图像错误 — 图像服务器不会替换其他字体。 此外，如果您要使用 **提取文本** 选项，在上载使用这些字体的PSD之前，必须上载字体文件。 此 **提取文本** 该功能将尝试将文本重新创建为可编辑的文本图层，并将其置于Dynamic Media Classic模板中。 这将在下一个主题PSD选项中讨论。

请注意，字体具有多个内部名称，这些名称通常与其外部文件名不同。 您可以在Dynamic Media Classic中看到该资源的“详细信息”页面中看到其所有不同名称。 以下是Adobe Caslon Pro Semibold字体的名称，列在Dynamic Media Classic的“元数据”选项卡下：

![图像](assets/basic-templates/basic-templates-4.jpg)

_Dynamic Media Classic中字体的“详细信息”页面上的“元数据”选项卡。_

Dynamic Media Classic使用此字体的文件名(ACaslonPro-Semibold)作为其资源ID，但此名称不是模板使用的名称。 该模板使用底部列出的富文本格式(RTF)名称。 RTF是图像服务器文本引擎的本机“语言”。

如果需要通过URL更改字体，则必须调用字体的RTF名称（而非资产ID），否则您将收到错误。 在这种情况下，此字体的正确名称将是“Adobe Caslon Pro”。 我们将在下面的“RTF和文本参数”主题中讨论有关字体和RTF的更多信息。

在Windows和Mac系统上最常见的字体文件格式是OpenType和TrueType。 OpenType的扩展名为.OTF，而TrueType的扩展名为.TTF。 这两种格式在Dynamic Media Classic中的效果相同。

### 上传PSD时选择选项

您无需上传Photoshop文件(PSD)即可创建模板；模板可以利用Dynamic Media Classic中的任何图像资源构建。 但是，上传PSD可以简化创作过程，因为通常情况下，这些资源已位于分层PSD中。 此外，在上传分层PSD时，Dynamic Media Classic会自动生成模板。

- **保留图层。** 这是最重要的选项。 这告知Dynamic Media Classic为每个Photoshop图层创建一个图像资源。 如果未选中，将禁用所有其他选项，并且该PSD将被拼合到单个图像中。
- **创建** **模板。** 此选项会采用各种生成的图层，并通过将它们组合在一起来自动创建模板。 使用自动生成的模板的一个缺点是，Dynamic Media Classic将所有图层放在一个文件中，而每个图层只需要一个占位符。 删除多余的图层已经很容易，但是如果您有许多图层，则重新创建它们会更快。 请务必重命名新模板；如果不重命名，则下次重新上传同一PSD时将覆盖新模板。
- **提取文本。** 这会将模板中的文本图层重新创建为PSD中的文本图层（使用您上载的字体）。 如果您的文本位于Photoshop中的路径上，并且您想要在模板中维护该路径，则需要执行此步骤。 此功能要求您使用 **创建模板** 选项，因为提取的文本只能由上载时生成的模板创建。
- **将图层扩展至背景大小。** 此设置使每个图层的大小与整个PSD画布的大小相同。 这对于始终保持固定位置的图层非常有用：否则，在将图像交换到同一图层时，可能必须重新定位它们。
- **图层命名。** 这可告知Dynamic Media Classic如何为每个层生成的每个资源命名。 我们建议执行以下任一操作 **Photoshop** **和图层** **名称** 或Photoshop和 **图层** **数字**. 这两个选项都使用PSD名称作为名称的第一部分，并在末尾添加图层名称或编号。 例如，如果您有一个名为“shirt.psd”的PSD，并且其中包含名为“front”、“sleeves”和“collar”的图层(如果您使用 **Photoshop和** 图层 **名称** 选项，Dynamic Media Classic将生成资产ID“shirt_front”、“shirt_sleeves”和“shirt_collar”。 使用其中一个选项有助于确保名称在Dynamic Media Classic中是唯一的。

## 创建带图像图层的模板

即使Dynamic Media Classic可以根据分层PSD自动创建模板，您也应该知道如何手动构建模板。 如上所述，在某些情况下，您不希望使用Dynamic Media Classic创建的模板。

### 模板基础UI

我们先熟悉一下编辑界面。

左中是显示最终模板预览的工作区。 右侧是“图层”和“图层属性”面板。 在这些领域，您所做的工作最多。

![图像](assets/basic-templates/basic-templates-5.jpg)

_“生成模板基础知识”页面。_

- **预览/工作区。** 这是主窗口。 在这里，您可以用鼠标移动、调整图层和旋转图层。 图层轮廓显示为虚线。
- **图层。** 这类似于Photoshop图层面板。 向模板中添加图层时，它们将显示在此处。 从上到下栈叠图层 — “图层”面板中的顶层图层显示在列表中其下方的其它图层的上方。
- **图层属性。** 在这里，您可以使用数字控件调整图层的所有属性。 首先选择一个图层，然后调整其属性。
- **复合** **URL。** UI底部是复合URL区域。 本教程的此部分不会讨论此问题，但在这里，您将看到模板被解构为一系列图像服务URL修饰符。 此区域可编辑 — 如果您非常熟悉图像服务器命令，可以在此处手动编辑模板。 但您也可以破坏它。 与Photoshop一样，图层编号从0开始。 画布是第0层，您自己添加的第一层是第1层。 混合模式确定图层的像素与其下方的像素混合的方式。 可以使用混合模式创建各种特殊效果。

#### 使用模板基础知识编辑器

以下是启动基本模板的工作流步骤：

1. 在Dynamic Media Classic中，转到 **构建>模板基础知识**. 您可以不选择任何内容，也可以选择图像作为模板的第一层。
2. 选择大小，然后按 **确定**. 此大小应该与您在Photoshop中设计的大小匹配。 将加载模板编辑器。
3. 如果在步骤1中未选择图像，请在左侧资源面板中搜索或浏览到图像，然后将其拖动到工作区。

   - 图像将自动调整大小以适合画布的大小。 如果您计划替换高分辨率图像，则通常会引入一个大型（2000像素）PTIFF图像并将其用作占位符。
   - 这应该是模板的最底层，但您可以稍后对这些层重新排序。

4. 直接在工作区域中调整图层大小或重新定位图层，或通过调整“图层属性”面板中的设置。
5. 根据需要拖入其他图像图层。 如果需要，还可以添加图层效果。 请参阅主题 _添加图层效果_，如下所示。
6. 单击 **保存**，选择一个位置，然后为模板提供一个名称。 您可以预览，但此时您的模板将与拼合的Photoshop图像完全相似 — 它尚不可更改。

### 添加图层效果

图像服务器支持一些程序化的图层效果 — 可更改图层内容外观的特殊效果。 它们的工作方式与Photoshop中的图层效果类似。 它们附着在某个层上，但不受该层控制。 您可以调整或移除它们，而无需对图层本身进行永久更改。

- **投影**. 在图层边界之外应用阴影，通过x和y像素偏移定位。
- **内阴影**. 在图层边界内应用阴影，通过x和y像素偏移定位。
- **外发光**. 在图层的所有边缘周围均匀地应用发光效果。
- **内发光**. 在图层的所有边缘内均匀地应用发光效果。

![图像](assets/basic-templates/basic-templates-6.jpg)

_包含和不包含投影的图层_

要添加效果，请单击 **添加效果**，然后从菜单中选择一种效果。 与普通图层一样，您可以在“图层”面板中选择一种效果，然后使用“图层属性”面板调整其设置。

阴影效果在水平或垂直方向上偏离图层，而“发光”效果在所有方向都均匀应用。 内部效果作用于图层的不透明部分的顶部，而外部效果仅影响透明区域。

详细了解[添加图层效果](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/creating-template.html#using-shadow-and-glow-effects-on-layers).

### 添加参数

如果所有操作都是组合图层并保存它们，则最终结果与拼合的Photoshop图像没有区别。 使模板具有特殊性的是将参数添加到每个层的属性的功能，以便可以通过URL动态更改这些参数。

在Dynamic Media Classic术语中，参数是可以链接到模板资产的变量，因此可以通过URL对其进行操作。 向图层添加参数时，Dynamic Media Classic会通过为参数名称添加美元符号($)来在URL中公开该属性 — 例如，如果您创建名为“size”的参数来更改图层的大小，Dynamic Media Classic将重命名您的参数$size。

如果不为资产添加参数，则该资产在Dynamic Media Classic数据库中保持隐藏状态，并且不会显示在URL中。

![图像](assets/basic-templates/parameters.png)

如果没有参数，URL通常会更长，尤其是如果您还使用动态文本时。 文本可为每个URL添加数十个额外的字符。

最后，初始参数集将成为模板中属性的默认值。 如果您构建模板、添加参数，然后调用URL而不调用其参数，则图像服务器将使用您在模板中保存的所有默认值创建图像。 只有在要更改属性时才需要参数。 如果不需要更改资产，则无需设置参数。

#### 创建参数

这是用于创建参数的工作流：

1. 单击 **参数** 按钮，该按钮位于要创建参数的层名称旁边。 将打开“参数”屏幕。 它列出了层上的每个属性及其值。
1. 选择 **日期** 选项，该选项位于要用作参数的每个资产的名称旁。 将显示默认参数名称。 您只能将参数添加到已从默认状态更改过的属性。

   - 例如，如果添加一个图层并将其保留在其默认xy位置0,0，则Dynamic Media Classic不会公开 **位置** 属性。 若要修复，请移动图层至少一个像素。 现在，Dynamic Media Classic将公开 **位置** 作为属性，您可以进行参数化。
   - 要向show/hide属性添加参数（打开和关闭图层），请单击 **显示** 或 **隐藏图层** 图标以关闭图层（如果需要，您可以稍后将其重新打开）。 Dynamic Media Classic现在将公开 **隐藏** 可参数化属性。

1. 将默认参数名称重命名为更易于在URL中识别的名称。 例如，如果要添加参数以更改图像顶部的横幅图层，请将默认名称“layer_2_src”更改为“banner”。
1. 按 **关闭** 以退出“参数”屏幕。
1. 通过单击 **参数** 按钮和添加及重命名参数。
1. 完成时保存更改。

>[!TIP]
>
>将参数重命名为有意义的名称，并开发命名约定来标准化这些名称。 确保设计和开发团队事先就命名约定达成一致。
>
>无法添加参数，因为看不到该属性？ 只需更改图层的属性，而不更改其默认属性（通过移动、调整大小、隐藏等）。 此时，您应会看到该资产已公开。

详细了解 [模板参数](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/creating-template-parameters.html).

## 创建带有文本图层的模板

现在，您将了解如何创建包含文本图层的基本模板。

### 了解动态文本

您现在知道如何使用图像图层构建基本模板。 对于许多应用程序来说，这都是您所需要的。 正如在上一个练习中所看到的，具有简单文本（例如“Sale”和“New”）的图层可以栅格化并被视为图像，因为其文本不需要更改。

但是，如果您需要：

- 添加标签以显示“25%折扣”，其中值25%为变量
- 添加一个文本标签，在图像上方显示产品名称
- 根据看到模板的国家/地区，将图层本地化为不同的语言

在这种情况下，您需要添加一些带有参数的动态文本图层，以控制文本和/或格式。

要创建文本，您需要上传一些字体 — 否则，Dynamic Media Classic将默认使用Arial。 字体还必须发布到图像服务器，否则当它尝试渲染使用该字体的任何文本时，将生成错误。

### RTF和文本参数

要使用“模板基础知识”工具向文本添加变量，您应该了解文本的呈现方式。 图像服务器使用Adobe文本引擎(与Photoshop和Illustrator使用的引擎相同)生成文本，并将其复合为最终图像中的图层。 为了与引擎通信，图像服务器使用RTF格式。

RTF是由Microsoft开发的一种文件格式规范，用于指定文档的格式。 它是大多数文字处理和电子邮件软件使用的标准标记语言。 如果您写入URL &amp;text=\b1 Hello，则图像服务器将以粗体类型生成带有“Hello”一词的图像，因为\b1是用于使文本变为粗体的RTF命令。

好消息是Dynamic Media Classic会为您生成RTF。 每当您在模板中键入文本并添加格式时，Dynamic Media Classic都会自动将RTF代码悄悄地写入模板中。 我们之所以提到它，是因为您直接向RTF本身添加参数，因此请务必对它稍加了解。

#### 创建文本图层

可通过以下两种方式在Dynamic Media Classic的模板中创建文本图层：

1. Dynamic Media Classic中的文本工具。 下面我们将讨论此方法。 模板基础知识编辑器提供了一个工具，可用于创建文本框、输入文本和设置文本格式。 Dynamic Media Classic会根据需要生成RTF，并将其放置在单独的图层中。
2. 提取文本（上载时）。 另一种方法是在Photoshop中创建文本图层，并将其保存在PSD中作为普通文本图层（而不是将其栅格化为图像图层）。 然后，将该文件上传到Dynamic Media Classic并使用 **提取文本** 选项。 Dynamic Media Classic将使用RTF命令将每个Photoshop文本图层转换为图像服务文本图层。 如果使用此方法，请确保首先将字体上载到Dynamic Media Classic，否则Dynamic Media Classic将在上载时替换默认字体，并且没有简单的方法可替换正确的字体。

### 文本编辑器

使用文本编辑器输入文本。 文本编辑器是一个WYSIWYG界面，允许您使用与Photoshop或Illustrator中类似的格式控件输入文本并设置其格式。

![图像](assets/basic-templates/basic-templates-9.jpg)

_模板基础知识文本编辑器。_

大部分工作都在 **预览** 选项卡，允许您输入文本并查看文本在模板中的显示方式。 此外， **来源** 选项卡，必要时用于手动编辑RTF。

一般工作流是使用 **预览** 按Tab键键入一些文本。

然后，选择文本并使用顶部的控件选择某些格式，如字体颜色、字体大小或对齐。 按所需的方式设置文本样式后，单击 **应用** 以在工作区预览中查看其更新。 然后，关闭文本编辑器以返回到“模板基础知识”主窗口。

#### 使用文本编辑器

以下是在“模板基础知识”构建页面中添加文本的工作流步骤：

1. 单击 **文本** “生成”页面顶部的工具按钮。
2. 将文本框拖出到要显示文本的位置。 将在模式窗口中打开文本编辑器窗口。 在背景中，您将看到模板，但在完成对文本的编辑之前，该模板不可编辑。
3. 键入要在首次加载模板时显示的示例文本。 例如，如果您要为个性化的电子邮件图像创建文本框，则文本可能会显示“您好！ 现在是时候拯救世界了！” 之后，您将添加一个文本参数来将“名称”替换为您在URL上发送的值。 在单击之前，您的文本不会显示在窗口下方的模板中 **应用**.
4. 要设置文本的格式，请使用鼠标拖动选择文本，然后在UI中选择格式控件。

   - 有许多格式选项。 其中最常见的是字体（字体）、字体大小和字体颜色，以及左对齐/居中/右对齐。
   - 不要忘记先选择文本。 否则，您将无法应用任何格式。
   - 要选择其他字体，请确保选择文本并打开“字体”菜单。 该编辑器将显示已上传到Dynamic Media Classic的所有字体的列表。 如果计算机上也安装了字体，则它将显示为黑色。 如果您的计算机上未安装该软件，则它将显示为红色。 但是，当您单击时，它仍将在预览窗口中呈现 **应用**. 您只需将字体上传到Dynamic Media Classic，任何使用Dynamic Media Classic的人都可以使用。 发布后，图像服务器将使用这些字体生成文本 — 您的用户无需安装任何字体即可查看您创建的文本，因为它是图像的一部分。
   - 与Photoshop和Illustrator不同，图像服务器可以在文本框中垂直对齐文本。 缺省值为顶部对齐。 要更改此内容，请选择您的文本并选择 **中间** 或 **底部** 从 **垂直对齐** 菜单。
   - 如果使文本对于文本框太大（或文本框太小），则全部或部分文本将被剪切并消失。 减小字体大小或增大框。

5. 单击 **应用** 以使所做的更改在工作区窗口中生效。 您必须单击 **应用**，否则将丢失您的编辑。
6. 完成后，单击 **关闭**. 如果要返回编辑模式，请双击文本图层以重新打开文本编辑器。

如果字体安装在本地系统上，文本编辑器将预览该字体到底有多大。

### 关于向文本图层添加参数

现在，我们按照与添加图层参数类似的过程来添加文本参数。 文本图层还可以采用大小、位置等图层参数；但是，它们也可以采用其他参数来控制RTF的任何方面。

与层参数不同，您只需选择要更改的值并向其中添加参数，而无需向整个属性添加参数。

![图像](assets/basic-templates/basic-templates-10.jpg)

示例RTF：

![图像](assets/basic-templates/sample-rtf.png)

在检查RTF时，您需要确定每个设置要更改的位置。 在上面的RTF中，某些格式可能比较合理，您可以看到格式来自何处。

你可以看到Chocolate Mint Sandal这个短语 — 这是文字本身。

- 这里有一个字体参考Por Richard — 这是选择字体的地方。
- 您可以看到RGB值：\red56\green53\blue4 — 这是文本颜色。
- 虽然字体大小为20，但您看不到数字20。 但是，您确实看到一个命令\fs40 — 出于一些奇怪的原因，RTF将字体测量为半点。 因此\fs40是字体大小！

您拥有足够的信息来创建参数，但是“图像服务”文档中包含所有RTF命令的完整引用。 访问 [图像服务文档](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/image-serving-api/image-serving-api/http-protocol-reference/text-formatting/c-text-formatting.html#concept-0d3136db7f6f49668274541cd4b6364c).

#### 向文本图层添加参数

以下是向文本图层添加参数的步骤。

1. 单击 **参数** 按钮（一个“P”）。 将打开“参数”屏幕。 此 **公共** 选项卡列出了层上的每个属性及其值。 您可以在此处添加常规图层参数。
1. 单击 **文本** 选项卡。 在这里，您可以看到顶部的RTF；您添加的参数位于其下方。
1. 要添加参数，首先加亮要更改的值，然后单击 **添加参数** 按钮。 确保只选择命令的值，而不是整个命令本身。 例如，要为上述示例RTF中的字体名称设置参数，我只会突出显示“Power Richard”并向其中添加参数，但不包括“\f0”。 当您单击 **添加参数** ，它显示在下面的列表中，并且参数值在仍处于选中状态时在RTF中以红色显示。 如果需要删除参数，请单击 **日期** 关闭该参数，该参数将消失。
1. 单击可将参数重命名为更有意义的名称。
1. 完成后，您的RTF将以绿色突出显示，其中参数存在，并且您的参数名称和值如下所列。
1. 单击 **关闭** 以退出“参数”屏幕。 然后按下 **保存** ，以保存模板。 如果您已完成编辑，请按 **关闭** 以退出“模板基础知识”页面。
1. 单击 **预览** 以在Dynamic Media Classic中测试模板。 要测试文本参数，请在预览窗口中键入新文本或新值。 要更改字体，必须键入该字体的确切RTF名称。

>[!TIP]
>
>若要向文本颜色添加参数，请分别添加红色、绿色和蓝色参数。 例如，如果RTF为 `\red56\green53\blue46`，则可以为值56、53和46添加单独的红色、绿色和蓝色参数。 在URL中，您可以通过调用以下三种方式来更改颜色： `&$red=56&$green=53&$blue=46`.

了解如何 [创建动态文本参数](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/creating-template-parameters.html#creating-dynamic-text-parameters).

## 发布和创建模板URL

### 创建图像预设

不需要为模板创建预设。 我们建议将此作为最佳实践，以便始终以1:1的大小调用模板，并且还要向调整大小以适合模板的任何大型图像图层添加锐化。 如果调用没有预设的图像，则图像服务器可能会将图像大小任意调整为默认大小（约400像素），并且不会应用默认锐化。

对于模板的图像预设，没有什么特别之处。 如果您已经为相同大小的静态图像设置了预设，则可以改用它。

### 发布

您需要运行发布以查看将更改实时推送到图像服务器。 请牢记需要发布的内容：各种图像资源图层、动态文本的字体以及模板本身。 与其他Dynamic Media Classic富媒体资产（如图像集和旋转集）类似，Basic Template是一种人工构建 — 它是数据库中使用一系列图像服务命令引用图像和字体的行项目。 因此，当您发布模板时，您只需更新图像服务器上的数据即可。

详细了解 [发布模板](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/publishing-templates.html).

### 模板URL

如前所述，基本模板具有与普通图像调用相同的基本URL语法。 模板通常具有更多修饰符（命令以&amp;号分隔），例如带有值的参数。 不过，主要区别在于您调用模板作为主图像，而不是调用静态图像。

![图像](assets/basic-templates/basic-templates-11.jpg)

图像预设在预设名称的两边各有一个美元符号($)，而参数在开头只有一个美元符号。 这些美元符号的放置非常重要。

**正确：**

`$text=46-inch LCD HDTV`

**不正确：**

`$text$=46-inch LCD HDTV`

`$text=46-inch LCD HDTV$`

`text=46-inch LCD HDTV`

如前所述，参数用于更改模板。 如果在没有参数的情况下调用模板，它将恢复为在“模板基础”创作工具中设计的默认设置。 如果不需要更改资产，则无需设置参数。

![图像](assets/basic-templates/sandals-without-with-parameters.png)
_未设置参数（上图）和参数（下图）的模板示例。_
