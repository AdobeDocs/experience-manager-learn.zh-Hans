# 基本模板简介{#basic-templates}

在Dynamic Media经典术语中，模板是一个文档，在模板发布后，可以通过URL动态更改。 Dynamic Media经典优惠基本模板，从图像服务器调用的基于图像的模板，由图像和渲染的文本组成。

模板最强大的方面之一是它们具有直接集成点，使您可以将它们绑定到数据库。 因此，您不仅可以提供一个图像并调整其大小，还可以查询数据库以查找新项目或销售项目，并在图像上显示为叠加。 您可以要求提供项目的说明，并使其在您选择的字体和布局中显示为标签。 可能性是无限的。

基本模板可以通过多种不同的方式实现，从简单到复杂。 例如：

- 基本销售。 如果产品有免费送货，则使用“免费送货”等标签。 这些标签由Photoshop中的商品团队设置，Web使用逻辑来知道何时将它们应用于图像。
- 高级销售。 每个模板具有多个变量，并且可能同时显示多个选项。 使用数据库、库存和业务规则确定何时将产品显示为“Just In”（刚进）、“Clearance”（清理）或“Sold Out”（售罄）。 还可以在产品后面使用透明度，在不同的背景（如不同的房间）上显示它。 相同的模板和/或资产可在产品详细信息页面上重新确定用途，以在不同背景上显示同一产品的更大或可缩放版本。

了解Dynamic Media Classic仅提供这些基于模板的应用程序的可视部分非常重要。 Dynamic Media Classic公司或其集成合作伙伴必须提供业务规则、数据库和开发技能来创建应用程序。 没有“内置”模板应用程序；设计人员在Dynamic Media Classic中设置模板，开发人员使用URL调用来更改模板中的变量。

在本教程的本节结尾，您将了解如何：

- 将Photoshop PSD上传到Dynamic Media Classic，以将其用作模板的基础。
- 创建由图像图层组成的简单促销基本模板。
- 添加文本图层，并通过使用参数使其变量。
- 构建模板URL并通过Web浏览器动态地处理图像。

>

## 基本模板概述

基本模板（或仅“模板”，简称为“模板”）的定义是URL可寻址的分层图像。 最终结果是一幅图像，但URL可以更改它。 它可以由照片、文本或图形组成，Dynamic Media Classic中的任意P-TIFF资源组合。

模板与Photoshop PSD文件最相似，因为它们具有相似的工作流程和类似的功能。

- 这两层都由类似于醋酸层叠的层组成。 您可以合成部分透明的图像，并通过图层的透明区域查看下面的图层。
- 可以移动和旋转图层以重新定位内容，并可以更改不透明度和混合模式以使内容部分透明。
- 您可以创建基于文本的图层。 图像服务器使用的文本引擎与Photoshop和Illustrator相同，因此质量可能非常高。
- 可以将简单的图层样式应用于每个图层以创建特殊效果，如投影或发光。

但是，与Photoshop PSD不同，图层可以完全动态，并通过图像服务器上的URL进行控制。

- 您可以向所有模板属性添加变量，以便更轻松地实时更改其合成。
- 称为参数的变量允许您仅公开要更改的模板部分。

您只需为每个将会改变的图层添加一个占位符，而不是像在Photoshop中那样将所有图层放入单个文件中，并显示和隐藏它们（尽管您也可以这样做，如果愿意）。

使用占位符，您可以将某个图层的内容与另一个已发布的资产动态交换，它将自动采用它所替换的图层的相同属性（如大小和旋转）。

由于基本模板通常在Photoshop中设计，但通过URL部署，因此模板项目既需要设计技能，又需要技术技能。 我们通常假定创作模板的人员是Photoshop设计人员，而实施模板的人员是Web开发人员。 创意团队和开发团队必须紧密合作，以使模板取得成功。

模板项目可能相对简单或极其复杂，具体取决于业务规则和应用程序的需求。 基本模板是从图像服务器中调用的，但是，由于Dynamic Media经典环境的灵活性，您甚至可以将模板嵌套在其他模板中，这样您就可以创建出相当复杂的图像，这些图像可以通过通常命名的变量链接。

- 了解有关[模板基础知识](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/quick-start-template-basics.html)的更多信息。
- 了解如何创建[基本模板](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/creating-template.html#creating_a_template)。

## 创建基本模板

使用基本模板时，通常按照下图中的工作流步骤操作。 如果您使用动态文本图层，则标记有虚线的步骤是可选的，并在下面的说明中指示为“文本工作流”。 如果不使用文本，则只遵循主路径。

![图像](assets/basic-templates/basic-templates-1.png)

_基本模板工作流。_

1. 设计和创建您的资源。 大多数用户在Adobe Photoshop中这样做。 以您需要的确切大小设计资源 — 如果缩略图页面的图像为200像素，则以200像素进行设计。 如果需要放大它，请将其设计为大约2000像素。 使用Photoshop(和/或另存为位图的Illustrator)创建资源，使用Dynamic Media Classic将各个部分组合到一起，管理图层并添加变量。
2. 设计图形资源后，将它们上传到Dynamic Media Classic。 我们建议您上传整个图层的PSD文件，而不是从PSD上传单个资产，而是使用上传时的&#x200B;**维护图层**&#x200B;选项，为每个图层创建一个文件（有关详细信息，请参阅下面的内容）。 _文本工作流：如果创建动态文本，请同时上传您的字体。动态文本是可变的，并通过URL进行控制。 如果您的文本是静态的，或只有几个短语没有更改（例如，说“New”或“Sale”的标记，而不是“X% Off”，而X是一个变量数），我们建议在Photoshop中预渲染文本，并将栅格化图层上传为图像。 这样会更简单，您可以根据需要完全设置文本样式。_
3. 使用“构建”菜单的“模板基础”编辑器在Dynamic Media Classic中构建模板，并添加图像图层。 文本工作流：在同一编辑器中创建文本图层。 在Dynamic Media Classic中手动构建模板时，需要执行此步骤。 选择与您的设计匹配的画布大小，将图像拖放到画布上，并设置图层属性（大小、旋转、不透明度等）。 您不会将每个可能的图层都放置在模板上，而只是每个图像图层一个占位符。 _文本工作流：使用文本工具创建文本图层，与在Photoshop中创建文本图层类似。您可以使用Photoshop文字工具中提供的相同选项选择字体并设置其样式。_ 另一个工作流是上传PSD并让Dynamic Media Classic生成“免费”模板，甚至可以重新创建文本图层。稍后将更详细地讨论此问题。
4. 创建图层后，向要通过URL控制的任何图层的任何属性(包括图层的源（图像本身）添加参数（变量）。 _文本工作流：您还可以向文本图层添加参数，既可以控制文本内容，也可以控制图层本身的大小和位置，也可以控制所有格式选项，如字体颜色、字号、水平跟踪等。_
5. 创建与模板大小匹配的图像预设。 我们建议这样做，以便始终以1:1大小调用模板，并且还要为任何大图像图层添加锐化功能，以便调整大小以适合模板。 如果要构建要放大的模板，则不必执行此步骤。
6. 发布，从Dynamic Media Classic预览复制URL，并在浏览器中测试它。

## 准备模板资产并将其上传到Dynamic Media Classic

在将模板资源上传到Dynamic Media Classic之前，您需要完成一些准备步骤。

### 准备要上传的PSD

在将Photoshop文件上传到Dynamic Media Classic之前，请简化Photoshop中的图层，以便更轻松地使用图像服务器并与图像服务器实现最大兼容。 您的PSD文件通常由Dynamic Media Classic无法识别的许多元素组成，您最终可能还会遇到许多难以管理的小部分。 请务必保存主控PSD的备份，以备以后编辑原始PSD时使用。 您将上传简化的副本，而不是主控。

![图像](assets/basic-templates/basic-templates-2.jpg)

1. 通过将需要打开/关闭的相关图层合并/拼合到单个图层中，简化图层结构。 例如，标签“NEW”和蓝色横幅将合并到单个图层中，这样您只需单击一下即可显示或隐藏它们。
   ![图像](assets/basic-templates/basic-templates-3.jpg)
2. 某些图层类型和图层效果不受Dynamic Media Classic或图像服务器支持，在上传之前需要栅格化。 否则，可能会忽略效果，或丢弃图层。 栅格化图层意味着从可编辑到不可编辑进行转换。 要栅格化图层效果或文本图层，请创建空图层，同时选择并使用&#x200B;**“图层”>“合并图层”**&#x200B;或CTRL + E/CMD + E合并。

   - Dynamic Media Classic无法对图层进行编组或链接。 组或链接集中的所有图层都将转换为不再分组/链接的单独图层。
   - 图层蒙版将在上传时转换为透明度。
   - 不支持调整图层，将丢弃调整图层。
   - 填充图层（如纯色图层）将栅格化。
   - 在上传时，智能对象图层和矢量图层将栅格化为普通图像，并应用和栅格化智能过滤器。
   - 除非使用“提取文本”选项，否则文本图层也将栅格化 — 有关详细信息，请参阅下面的内容。
   - 大多数图层效果将被忽略，只支持少数混合模式。 如果有疑问，请在Dynamic Media Classic中添加简单效果（如内阴影或投影、内阴影或外发光），或使用空白图层在Photoshop中合并和栅格化效果。

### 使用字体

如果需要生成动态文本，您还将上传和发布字体。 Dynamic Media Classic附带的唯一字体是Arial。

每个公司都有责任获取在Web上使用字体的许可证 — 只是在您的计算机上安装字体并不赋予您在Web上以商业方式使用它的权利，并且如果未经许可而使用，您的公司可能会面临字体发布者的法律诉讼。 此外，许可条款也会有所不同，例如，您可能需要单独的许可才能进行打印，而不是屏幕显示。

Dynamic Media Classic支持标准OpenType(OTF)、TrueType(TTF)和Type 1 Postscript字体。 不支持仅Mac的手提箱字体、文字集合文件、Windows系统字体和专有机器字体（如雕刻或刺绣机使用的字体） — 您必须将它们转换为一种标准字体格式，或替换在Dynamic Media Classic和图像服务器上使用的类似字体。

在将字体上传到Dynamic Media Classic（与任何其他资源一样）后，这些字体还必须发布到图像服务器。 一个非常常见的模板错误是忘记发布字体，这会导致图像错误 — 图像服务器不会替换其它字体。 此外，如果要在上传时使用&#x200B;**“提取文本**”选项，则必须先上传您的字体文件，然后才能上传使用这些字体的PSD。 **提取文本**&#x200B;功能将尝试将文本重新创建为可编辑文本图层，并将其放置在Dynamic Media Classic模板中。 在下一个主题PSD选项中将讨论此问题。

请注意，字体具有多个内部名称，这些内部名称通常与其外部文件名不同。 在Dynamic Media Classic中，您可以在该资产的“详细信息”页面上查看其所有不同名称。 以下是Adobe Caslon Pro Semibold字体的名称，列在Dynamic Media Classic的“元数据”选项卡下：

![图像](assets/basic-templates/basic-templates-4.jpg)

_Dynamic Media Classic中某个字体的“详细信息”页面上的“元数据”选项卡。_

Dynamic Media Classic将此字体的文件名(ACaslonPro-Semibold)用作其资源ID，但该名称不是模板使用的名称。 模板使用底部列出的富文本格式(RTF)名称。 RTF是图像服务器文本引擎的本机“语言”。

如果需要通过URL更改字体，则必须调用字体的RTF名称（而非资产ID），否则您将收到错误。 在这种情况下，此字体的正确名称为“Adobe Caslon Pro”。 我们将在下面的主题“RTF和文本参数”中讨论有关字体和RTF的更多信息。

在Windows和Mac系统上，最常见的字体文件格式是OpenType和TrueType。 OpenType具有.OTF扩展名，而TrueType为.TTF。 两种格式在Dynamic Media Classic中同样有效。

### 上传PSD时选择选项

您无需上传Photoshop文件(PSD)即可创建模板；可以使用Dynamic Media Classic中的任何图像资源构建模板。 但是，上传PSD可以使创作更加轻松，因为您通常将这些资源已包含在图层式PSD中。 此外，Dynamic Media Classic在您上传图层式PSD时将自动生成模板。

- **保持图层。** 这是最重要的选择。这告诉Dynamic Media Classic每个Photoshop图层创建一个图像资源。 如果未选中，则会禁用所有其他选项，并将PSD拼合到单个图像中。
- **创** **建模板。** 此选项会采用生成的各种图层，并通过将它们组合在一起自动创建模板。使用自动生成的模板的缺点是Dynamic Media Classic将所有图层放在一个文件中，而我们只需要每个图层一个占位符。 删除额外的图层非常简单，但如果您有许多图层，则重新创建它们会更快。 请务必重命名新模板；如果不这样做，下次重新上传同一PSD时，它将被覆盖。
- **提取文本。** 这会使用您上传的字体将PSD中的文本图层重新创建为模板中的文本图层。如果文本位于Photoshop中的路径中，并且要在模板中保持该路径，则需要执行此步骤。 此功能要求您使用&#x200B;**创建模板**&#x200B;选项，因为提取的文本只能由上载时生成的模板创建。
- **将图层扩展到背景大小。** 此设置使每个图层的大小与整个PSD画布相同。对于始终保持固定位置的图层，此选项非常有用：否则，在将图像交换到同一图层时，您可能必须重新定位它们。
- **图层命名。** 这告诉Dynamic Media Classic如何为每个图层生成的每个资源命名。我们建议使用&#x200B;**Photoshop** **和Layer** **Name**&#x200B;或Photoshop和&#x200B;**Layer** **Number**。 这两个选项都使用PSD名称作为名称的第一部分，并在末尾添加图层名称或编号。 例如，如果您有一个名为“shirt.psd”的PSD，并且它有名为“front”、“silfes”和“collar”的图层，则如果使用&#x200B;**Photoshop和** Layer **Name**&#x200B;选项上传，Dynamic Media Classic将生成资产ID“st_front”、“s”、“shirt_sit_s”、“sits”、“shirt_sifes”、“s”、“s”、“sits”和“s”。&quot; 使用其中一个选项有助于确保名称在Dynamic Media Classic中是唯一的。

## 创建包含图像图层的模板

即使Dynamic Media Classic可以根据分层的PSD自动创建模板，您也应知道如何手动构建模板。 如上所述，有时您不想使用由Dynamic Media Classic创建的模板。

### 模板基础UI

让我们首先熟悉一下编辑界面。

左中心是您的工作区，显示您的最终模板的预览。 右侧是“图层”和“图层属性”面板。 这些领域是您最擅长的领域。

![图像](assets/basic-templates/basic-templates-5.jpg)

_构建模板基础知识页面。_

- **预览/工作区。** 这是主窗。在此，您可以使用鼠标移动、调整图层大小和旋转图层。 图层轮廓显示为虚线。
- **图层。** 这与Photoshop图层面板类似。在向模板中添加图层时，图层将显示在此。 图层从上到下堆叠 — “图层”面板中的顶层图层将显示在“图层”面板中其下的其他图层上方。
- **图层属性。** 您可以在此使用数字控件调整图层的所有属性。首先选择一个图层，然后调整其属性。
- **** **CompositeURL。** UI底部是复合URL区域。本教程的本节将不讨论此问题，但您将在此看到模板被解构为一系列图像服务URL修饰符。 此区域是可编辑的 — 如果您非常熟悉图像服务器命令，则可以在此处手动编辑模板。 但您也可以打破它。 与Photoshop一样，图层编号开始为0。 画布为0层，您自己添加的第一个图层为1层。 混合模式确定图层的像素如何与其下方的像素混合。 您可以使用混合模式创建各种特殊效果。

#### 使用模板基础编辑器

以下是开始基本模板的工作流步骤：

1. 在Dynamic Media Classic中，转至&#x200B;**构建>模板基础**。 您可以选择未选择任何内容，也可以通过选择将成为模板第一层的图像来开始。
2. 选择“Size（大小）” ，然后按&#x200B;**“OK**。 此大小应与您在Photoshop中设计的大小相匹配。 将加载模板编辑器。
3. 如果在第1步中未选择图像，请在左侧的“资源”面板中搜索或浏览到图像，然后将其拖动到工作区域。

   - 图像将自动调整大小以适合画布的大小。 如果您计划替换高分辨率图像，则通常会导入一张大（2000像素）P-TIFF图像并将其用作占位符。
   - 这应是模板的最底层图层，但您以后可以对图层重新排序。

4. 直接在工作区中或通过调整“图层属性”面板中的设置来调整图层大小或重新放置图层。
5. 根据需要拖入其他图像图层。 如果需要，还可添加图层效果。 请参阅下面的主题&#x200B;_添加图层效果_。
6. 单击&#x200B;**保存**，选择一个位置，然后为模板指定一个名称。 您可以预览，但此时模板将完全像拼合的Photoshop图像 — 它尚不可更改。

### 添加图层效果

图像服务器支持一些程序化图层效果 — 可更改图层内容外观的特殊效果。 它们的工作方式与Photoshop中的图层效果类似。 它们附加到一个图层，但独立于该图层进行控制。 您可以调整或删除图层，而无需对图层本身进行永久更改。

- **投影**。在图层边界外部应用阴影，该阴影由x和y像素偏移定位。
- **内阴影**。在图层边界内应用阴影，通过x和y像素偏移定位。
- **外发光**。在图层的所有边缘周围均匀地应用发光效果。
- **内发光**。均匀地在图层的所有边缘内应用发光效果。

![图像](assets/basic-templates/basic-templates-6.jpg)

_具有和没有投影的图层_

要添加效果，请单击&#x200B;**添加效果**，然后从菜单中选择效果。 与普通图层一样，您可以在“图层”面板中选择效果，然后使用“图层属性”面板调整其设置。

阴影效果在水平或垂直方向上偏离图层，而发光效果在所有方向上均匀应用。 内效果作用在图层的不透明部分上方，而外效果仅影响透明区域。

了解有关[添加图层效果](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/creating-template.html#using-shadow-and-glow-effects-on-layers)的更多信息。

### 添加参数

如果您只是合并图层并保存它们，则最终结果与拼合的Photoshop图像无异。 模板的特殊之处在于能够向每个图层的属性添加参数，以便通过URL动态更改它们。

在Dynamic Media Classic术语中，参数是可链接到模板属性的变量，因此可以通过URL操纵它。 当您向图层添加参数时，Dynamic Media Classic会在URL中公开该属性，方法是在参数名称前加上一个美元符号($) — 例如，如果您创建一个名为“size”的参数来更改图层的大小，Dynamic Media Classic会重命名您的参数$size。

如果未为属性添加参数，则该属性仍会隐藏在Dynamic Media Classic数据库中，而不会显示在URL中。

![图像](assets/basic-templates/parameters.png)

如果没有参数，URL通常会更长，尤其是如果您还使用动态文本。 文本会在每个URL上添加许多额外字符。

最后，初始参数集将成为模板中属性的默认值。 如果您构建模板，添加参数，然后调用URL（不包含其参数），图像服务器将创建包含您在模板中保存的所有默认值的图像。 仅当要更改属性时才需要参数。 如果某个属性不需要更改，则无需设置参数。

#### 创建参数

这是用于创建参数的工作流：

1. 单击要为其创建参数的图层名称旁边的&#x200B;**参数**&#x200B;按钮。 “参数”(Parameters)屏幕将打开。 它列表图层上的每个属性及其值。
1. 在要作为参数的每个属性的名称旁边选择&#x200B;**On**&#x200B;选项。 将显示默认参数名称。 您只能向从其默认状态更改的属性添加参数。

   - 例如，如果添加一个图层并将其保持在默认的xy位置0,0,Dynamic Media Classic将不显示&#x200B;**Position**&#x200B;属性。 要修复，请至少移动一个像素。 现在，Dynamic Media Classic会将&#x200B;**Position**&#x200B;作为可参数化的属性进行显示。
   - 要向show/hide属性添加参数（这会打开和关闭图层），请单击&#x200B;**显示**&#x200B;或&#x200B;**隐藏图层**&#x200B;图标以关闭图层（如果需要，可以随后将其重新打开）。 Dynamic Media Classic现在将公开可进行参数化的&#x200B;**Hide**&#x200B;属性。

1. 将默认参数名称重命名为在URL中更容易识别的名称。 例如，如果要添加参数以更改图像顶部的横幅图层，请将默认名称“layer_2_src”更改为“banner”。
1. 按&#x200B;**关闭**&#x200B;退出“Parameters（参数）”屏幕。
1. 通过单击&#x200B;**Parameters**&#x200B;按钮并添加和重命名参数，对其他图层重复此过程。
1. 完成后保存更改。

>[!TIP]
>
>将您的参数重命名为有意义的内容，并制定命名约定以标准化这些名称。 确保设计和开发团队事先商定了命名约定。
>
>无法添加参数，因为您看不到该属性？ 只需将图层的属性从其默认值（通过移动、调整大小、隐藏等）更改。 您现在应该可以看到该属性。

了解有关[模板参数](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/creating-template-parameters.html)的更多信息。

## 创建带有文本图层的模板

现在，您将学习如何创建包含文本图层的基本模板。

### 了解动态文本

您现在了解如何使用图像图层构建基本模板。 对于许多应用程序，这是您所需的全部。 正如您在上一练习中所看到的，具有简单文本（如“Sale”和“New”）的图层可以栅格化并作为图像处理，因为它们的文本无需更改。

但是，如果您需要：

- 添加一个标签，说“25%优惠”，其值为25%可变
- 在图像顶部添加带有产品名称的文本标签
- 根据您看到模板所在的国家/地区，将您的图层本地化为不同的语言

在这种情况下，您需要添加一些包含参数的动态文本图层以控制文本和/或格式。

要创建文本，您需要上传一些字体 — 否则，Dynamic Media Classic将默认为Arial。 字体还必须发布到图像服务器，否则当字体尝试呈现使用该字体的任何文本时，该字体将生成错误。

### RTF和文本参数

要使用模板基础工具向文本添加变量，您应了解文本的呈现方式。 图像服务器使用Adobe文本引擎(Photoshop和Illustrator使用的相同引擎)生成文本，并将其合成为最终图像中的图层。 为了与引擎通信，图像服务器使用RTF格式。

RTF是Microsoft开发的用于指定文档格式的文件格式规范。 它是大多数文字处理和电子邮件软件使用的标准标记语言。 如果写入URL &amp;text=\b1 Hello，图像服务器将生成一个带有粗体文字字“Hello”字样的图像，因为\b1是用于使文本变粗的RTF命令。

好消息是Dynamic Media Classic会为您生成RTF。 在模板中键入文本并添加格式时，Dynamic Media Classic会自动将RTF代码静默地写入模板中。 我们提到此问题的原因是，您将直接向RTF中添加参数，因此，您应该对它有点熟悉。

#### 创建文本图层

您可以通过以下两种方式在Dynamic Media Classic中的模板中创建文本图层：

1. Dynamic Media Classic中的文本工具。 下面我们将讨论此方法。 模板基础编辑器中有一个工具，可让您创建文本框、输入文本和设置文本格式。 Dynamic Media Classic根据需要生成RTF并将其放置在单独的图层中。
2. 提取文本（上载时）。 另一种方法是在Photoshop中创建文本图层，并将其存储在PSD中作为普通文本图层（而不是将其栅格化为图像图层）。 然后，将文件上传到Dynamic Media Classic，并使用&#x200B;**提取文本**&#x200B;选项。 Dynamic Media Classic将使用RTF命令将每个Photoshop文本图层转换为图像服务文本图层。 如果您使用此方法，请确保首先将字体上传到Dynamic Media Classic，否则Dynamic Media Classic将在上传时替换默认字体，并且无法轻松地重新替换正确的字体。

### 文本编辑器

您可以使用文本编辑器输入文本。 文本编辑器是WYSIWYG界面，允许您使用与Photoshop或Illustrator中类似的格式控制输入文本并设置其格式。

![图像](assets/basic-templates/basic-templates-9.jpg)

_模板基础知识文本编辑器。_

您将在&#x200B;**预览**&#x200B;选项卡中完成大部分工作，该选项卡允许您输入文本并查看文本在模板中的显示效果。 还有一个&#x200B;**源**&#x200B;选项卡，用于根据需要手动编辑RTF。

一般工作流是使用&#x200B;**预览**&#x200B;选项卡键入某些文本。

然后选择文本，并使用顶部的控件选择一些格式，如字体颜色、字体大小或对齐。 在文本按您需要的方式设置样式后，单击&#x200B;**应用**&#x200B;以在工作区预览中查看其更新。 然后，关闭文本编辑器以返回到模板基础主窗口。

#### 使用文本编辑器

以下是在“模板基础知识”构建页面中添加文本的工作流步骤：

1. 单击构建页面顶部的&#x200B;**文本**&#x200B;工具按钮。
2. 拖出要显示文本的文本框。 “文本编辑器”(Text Editor)窗口将在模态窗口中打开。 在后台，您将看到模板，但只有编辑完文本后，模板才可编辑。
3. 键入首次加载模板时要显示的示例文本。 例如，如果您正在为个性化的电子邮件图像创建文本框，则您的文本可能会说“您好。 现在是省钱的时候了！” 稍后，您将添加一个文本参数，用您在URL上发送的值替换“名称”。 在单击&#x200B;**应用**&#x200B;之前，您的文本将不会显示在窗口下方的模板中。
4. 要设置文本的格式，请使用鼠标拖动以选择文本，然后在UI中选择格式控件。

   - 有许多格式选项。 一些最常见的字体（面）、字体大小和字体颜色以及左/中/右对齐。
   - 不要忘记先选择文本。 否则，您将无法应用任何格式。
   - 要选择其他字体，请确保选择文本并打开“字体”菜单。 编辑器将显示上传到Dynamic Media Classic的所有字体的列表。 如果计算机上也安装了字体，该字体将以黑色显示。 如果计算机上未安装，将以红色显示。 但是，单击&#x200B;**应用**&#x200B;时，它仍将在预览窗口中呈现。 您只需将字体上传到Dynamic Media Classic，即可让使用Dynamic Media Classic的任何人使用它们。 发布后，图像服务器将使用这些字体生成文本 — 您的用户无需安装任何字体即可查看您创建的文本，因为它是图像的一部分。
   - 与Photoshop和Illustrator不同，图像服务器可以在文本框中垂直对齐文本。 默认为顶部对齐。 要更改此项，请选择文本，然后从&#x200B;**垂直对齐**&#x200B;菜单中选择&#x200B;**中间**&#x200B;或&#x200B;**底部**。
   - 如果文本太大，而不适合该框（或文本框太小），则全部或部分文本将被剪切并消失。 减小字体大小或增大框。

5. 单击&#x200B;**应用**，查看您的更改在工作区窗口中生效。 必须单击&#x200B;**应用**，否则将丢失编辑。
6. 完成后，单击&#x200B;**关闭**。 如果要返回编辑模式，请多次单击文本图层以重新打开文本编辑器。

文本编辑器将准确预览字体大小（如果您的系统上已本地安装字体）。

### 关于向文本图层添加参数

现在，我们按照类似的过程添加文本参数，就像我们对图层参数所做的一样。 文本层还可以采用图层参数来调整其大小、位置等；但是，它们可以采用允许您控制RTF任何方面的其他参数。

与图层参数不同，您只选择要更改的值并向该值添加参数，而不是向整个属性添加参数。

![图像](assets/basic-templates/basic-templates-10.jpg)

RTF示例：

![图像](assets/basic-templates/sample-rtf.png)

检查RTF时，需要确定要更改的每个设置。 在上面的RTF中，部分内容可能有一定意义，您可以看到格式设置的来源。

你可以看到“巧克力薄荷沙豆”这个词本身就是文字。

- 引用了Poor Richard字体 — 这是选择字体的位置。
- 您可以看到一个RGB值：\red56\green53\blue4  — 这是文本颜色。
- 虽然字号为20，但您看不到数字20。 但是，您确实会看到\fs40命令 — 由于某些奇怪的原因，RTF将字体测量为半点。 因此\fs40是字体大小！

您有足够的信息来创建参数，但是图像服务文档中提供了所有RTF命令的完整参考。 访问[图像服务文档](https://docs.adobe.com/content/help/en/dynamic-media-developer-resources/image-serving-api/image-serving-api/http-protocol-reference/text-formatting/c-text-formatting.html#concept-0d3136db7f6f49668274541cd4b6364c)。

#### 向文本图层添加参数

以下是向文本图层添加参数的步骤。

1. 单击要为其创建参数的文本图层名称旁边的&#x200B;**参数**&#x200B;按钮(“P”)。 “参数”(Parameters)屏幕将打开。 **Common**&#x200B;选项卡列表图层上的每个属性及其值。 您可以在此添加常规图层参数。
1. 单击&#x200B;**文本**&#x200B;选项卡。 这里，您可以看到顶部的RTF;您添加的参数将位于该参数的下方。
1. 要添加参数，请首先突出显示要更改的值，然后单击&#x200B;**添加参数**&#x200B;按钮。 确保仅选择命令的值，而不是整个命令本身。 例如，要在上面的示例RTF中为字体名称设置参数，我只会突出显示“Poor Richard”并在其中添加一个参数，但不会添加“\f0”。 单击&#x200B;**添加参数**&#x200B;时，该参数将显示在下面的列表中，在仍选中参数时，该参数值将以红色显示在RTF中。 如果需要删除参数，请单击&#x200B;**On**&#x200B;旁边的复选框以关闭该参数，该参数将消失。
1. 单击可将参数重命名为更有意义的名称。
1. 完成后，RTF将以绿色突出显示，其中存在参数，您的参数名称和值将列在下面。
1. 单击&#x200B;**关闭**&#x200B;退出“参数”屏幕。 然后按&#x200B;**保存**&#x200B;以保存模板。 如果编辑完成，请按&#x200B;**关闭**&#x200B;退出“模板基础知识”页面。
1. 单击&#x200B;**预览**&#x200B;以在Dynamic Media Classic中测试模板。 要测试文本参数，请在“预览”窗口中键入新文本或新值。 要更改字体，必须键入字体的确切RTF名称。

>[!TIP]
>
>要向文本颜色添加参数，请分别添加红色、绿色和蓝色的参数。 例如，如果RTF为`\red56\green53\blue46`，则应为值56、53和46添加单独的红、绿和蓝参数。 在URL中，您可以通过调用以下三种更改颜色：`&$red=56&$green=53&$blue=46`。

了解如何[创建动态文本参数](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/creating-template-parameters.html#creating-dynamic-text-parameters)。

## 发布和创建模板URL

### 创建图像预设

为模板创建预设不是必需步骤。 我们建议将此作为最佳实践，以便始终以1:1大小调用模板，并且还可以向任何大图像图层添加锐化，这些图层会调整大小以适合模板。 如果调用的图像时没有预设，图像服务器可以随意调整图像大小至默认大小（约400像素），而不会应用默认锐化。

模板的图像预设没有什么特别之处。 如果您已经为具有相同大小的静态图像设置了预设，则可以改用它。

### 发布

您需要运行发布才能查看将更改实时推送到图像服务器。 请记住需要发布的内容：各种图像资源图层、动态文本的字体以及模板本身。 与图像集和旋转集等其他Dynamic Media Classic富媒体资源类似，基本模板是人工构建 — 它是数据库中的一个行项，它使用一系列图像服务命令引用图像和字体。 因此，发布模板时，您所做的只是更新图像服务器上的数据。

了解有关[发布模板的更多信息](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/publishing-templates.html)。

### 模板URL构建

基本模板与常规图像调用具有相同的基本URL语法，如前所述。 模板通常具有更多修饰符(用&amp;符号(&amp;)分隔的命令)，如带值的参数。 但主要区别在于，您调用模板作为主图像，而不是调用静态图像。

![图像](assets/basic-templates/basic-templates-11.jpg)

图像预设在预设名称的每侧都有一个美元符号($)，而参数在开头有一个美元符号。 这些美元信号的放置很重要。

**正确：**

`$text=46-inch LCD HDTV`

**不正确：**

`$text$=46-inch LCD HDTV`

`$text=46-inch LCD HDTV$`

`text=46-inch LCD HDTV`

如前所述，参数用于更改模板。 如果调用模板时没有参数，该模板将恢复为在模板基础创作工具中设计的默认设置。 如果某个属性不需要更改，则无需设置参数。

![图](assets/basic-templates/sandals-without-with-parameters.png)
_像不设置参数（上图）和参数（下图）的模板示例。_
